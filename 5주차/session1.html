1. 에지 연결 (Edge Linking)

[수업 내용 요약]

  • 에지 연결: 검출된 불연속적인 에지 픽셀들을 유사한 특징(크기, 방향)을 가진 이웃끼리 묶어 의미 있는 경계선으로 만드는 과정입니다.
  • 연결 기준 (Local Processing): 두 픽셀 \(s\)와 \(t\)가 다음 두 조건을 모두 만족하면 연결된 것으로 간주합니다.
    1. 에지 강도(Magnitude) 유사도: \(|M(s) - M(t)| \le T_m\)
    2. 에지 방향(Angle) 유사도: \(|\alpha(s) - \alpha(t)| \le T_a\)
  • 본 시뮬레이션은 8-이웃(8-neighbors) 탐색을 통해 위 조건을 만족하는 픽셀 간에 연결선(빨간색)을 생성합니다.

슬라이더를 조절하여 에지 강도 임계값(\(T_m\))과 방향 임계값(\(T_a\))에 따라 픽셀들이 어떻게 연결되는지 확인해보세요. 각 픽셀 내의 화살표는 그래디언트 방향(\(\alpha\))을 나타냅니다.

2. 선분 근사 (Edge Approximation / RDP Algorithm)

[수업 내용 요약]

  • 에지 결과의 근사: 복잡한 곡선 형태의 에지 픽셀 군집을 여러 개의 단순한 '직선(선분)'들로 나누어 근사(Approximation)하는 RDP(Ramer-Douglas-Peucker) 과정입니다.
  • 분할 기준(Threshold \(h\)): 양 끝점을 연결한 선분 \(ax + by + c = 0\) 을 기준으로, 곡선 상의 점들 \((x_i, y_i)\) 중 직선과 가장 멀리 떨어진 수직 거리 \(h\)를 구합니다.
  • 이 최대 거리 \(h\)가 설정한 임계값보다 크면 해당 점을 쪼개어(Split) 선분을 분할하며, 거리가 임계값 이내가 될 때까지 이 과정을 재귀적으로 반복합니다.

슬라이더를 움직여 임계값에 따라 선분 근사가 어떻게 실시간으로 변하는지 확인해보세요. 캔버스 위에 마우스를 드래그하여 새로운 곡선을 덮어 그릴 수도 있습니다. (초록색 선: 기준이 되는 근사 선분)

3. 허프 변환 (Hough Transform)

[수업 내용 요약]

  • 허프 변환: 영상 공간 \((x, y)\)에 존재하는 선분이나 곡선의 파라미터를 파라미터 공간 \((\rho, \theta)\)으로 매핑하여 직선을 검출하는 기법입니다.
  • 영상 공간에서의 한 점은 다음 수식에 의해 파라미터 공간에서 사인(sine) 곡선으로 나타납니다.
    $$\rho = x \cos \theta + y \sin \theta$$
  • 수많은 에지 픽셀들이 만들어내는 곡선들이 가장 많이 교차하는 점(Accumulator 최댓값)이 원래 영상 공간에서의 가장 뚜렷한 직선을 의미합니다.

영상 공간 \((x, y)\)의 점이 파라미터 공간 \((\rho, \theta)\)에서 사인 곡선으로 어떻게 매핑되는지 확인합니다.

4. RANSAC (RANdom SAmple Consensus)

[수업 내용 요약]

  • RANSAC: 노이즈(Outlier)가 심하게 섞인 데이터 속에서 정상적인 데이터(Inlier)들을 찾아내어 수학적 모델(직선 등)을 예측하는 기법입니다.
  • 연산 과정:
    1. 무작위로 수학적 모델을 구성하기 위한 최소한의 점(직선의 경우 2개)을 뽑아 모델을 세웁니다.
      $$\text{직선의 방정식: } ax + by + c = 0$$
    2. 생성된 직선과 모든 데이터 포인트 \((x_i, y_i)\) 와의 거리를 계산하여 오차 허용 범위(Threshold) 내에 있는 정상 데이터(Inlier)를 카운트합니다.
      $$\text{점과 직선 사이의 거리: } d = \frac{|a x_i + b y_i + c|}{\sqrt{a^2 + b^2}} < \text{Threshold}$$
    3. 이 과정을 반복하며 가장 많은 Inlier를 포함한 모델이 최적의 모델(Best Model)로 선정됩니다.

노이즈(Outlier)가 많은 환경에서 직선을 어떻게 찾아내는지 확인합니다.
[1회 무작위 실행] 버튼을 눌러 점을 하나씩 뽑아보거나, [자동 실행]을 통해 최적의 모델을 찾아보세요.
