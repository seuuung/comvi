# 프로젝트 절대 규칙 (Project Absolute Rules)

이 문서는 본 프로젝트 코드를 유지보수하거나 확장할 때 **반드시 지켜야 할 절대 규칙**을 명시합니다.

## 1. MathJax 수식 작성 규칙 (절대 규칙)

본 시뮬레이터 웹사이트는 `MathJax 3`를 이용해 수학 연산식 및 LaTex를 렌더링하고 있습니다.
수식 문법이 깨지는 현상(화면 상에 기호가 그대로 노출되는 버그)을 방지하기 위해 다음 형태를 엄격히 준수해야 합니다.

### 금지 사항 (Do NOT Use)
* 인라인 수식(Inline Math) 작성 시 단일 달러 기호(`$`)를 구분자로 사용하지 마십시오.
  * ❌ 잘못된 예: `$(x_i, y_i)$`
  * ❌ 잘못된 예: `$h = 20$`

### 권장 사항 및 원칙 (Must Use)
1. **인라인 수식 (Inline Math)**
   일반 텍스트와 섞어서 한 줄에 수식을 작성할 때는 **반드시 `\(` 와 `\)` 로 감싸야 합니다.**
   * ✅ 올바른 예: `\((x_i, y_i)\)`
   * ✅ 올바른 예: `\(h = 20\)`

2. **블록 수식 (Display/Block Math)**
   새로운 줄에 수식을 중앙 정렬하여 크게 배치할 때는 **이중 달러 기호(`$$`)를 사용**합니다.
   * ✅ 올바른 예: `$$ \rho = x \cos \theta + y \sin \theta $$`

3. **동적 렌더링 (JavaScript innerHTML)**
   `script.js` 내부에서 슬라이더 이벤트 등 DOM 요소가 1초에도 수십 번씩 비동기 업데이트(innerHTML 교체)되는 UI 구간에서는 성능(Performance) 저하 방지를 위해 **MathJax 수식을 아예 사용하지 않는 것**을 원칙으로 합니다.
   * 대신 HTML 표준 태그(예: `<i>h</i> = 20px`) 문자열로만 구성하여 퍼포먼스 드롭을 예방해야 합니다.

4. **검증은 수동으로 합니다**
에이전트가 직접 브라우저를 열어 검증하지 않고 사용자에게 검증을 요청해야합니다
